Making Sensors Graphs

All the sensor data that we collect can be used to show real-time sensor information on the screen. However, apart from showing this real-time info in gauges, sliders or other graphics, often sensor data makes most sense if users see trend lines over time.

For this purpose, I included the rrdtool library into LamPI.

 

Installing rrdtool

Before being able to define or even show sensor data in graphs, rrdtool and driver software needs to be loaded to the system.

sudo apt-get -y install python-smbus
sudo apt-get -y install python-dev

 

 

The graphing process

If all rrd software is installed, all sensors are working and we have a process in place to get sensor data in some form available by calling a script, then we are ready for the next steps:

  • Extract the sensor values from one or more executables
    • /home/pi/rrd/scripts/bmp085_extract.py
    • /home/pi/rrd/scripts/ds18b20_extract.sh

  • Update the rrdtool databases with these values
    The update script is called to updatethe rrdtool databases with values measured by our sensors.
    • /home/pi/rrd/scripts/update.sh

  • Generate graph files (.png) to show on our website
    One or several files can be defined that convert the data in .rrd files into graphics file.
    • /home/pi/rrd/scripts/generate.sh
